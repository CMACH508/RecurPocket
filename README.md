# RecurPocket: Recurrent Lmser Network with Gating Mechanism for Protein Binding Site Detection

In this paper, we propose a recurrent Lmser network, called RecurPocket, to improve the binding site prediction performance. Protein binding site detection is the basis of new drug discovery and design, but existing methods have low accuracy with incomplete prediction and impurities. So to address the challenges we propose RecurPocket to improve the feature embedding of protein structure for accurate binding site prediction in a recurrent and progressive manner. We derive not only forward skip connections from the encoder to decoder, but also feedback links in the opposite direction to reduce the deviation in encoder and guide the representation learning on the relevant structure in the next round. A binary voxel-level mask generated by the result of the previous round of decoder closes to the predicted contour can remove the noisy patterns which may be possibly induced by the feedback connections. Experiments on benchmark datasets demonstrate that RecurPocket is superior to state-of-the-art methods in predicting binding site.

## Dataset Preprocessing
#### 1. train dataset and test dataset
In our experiments, six different datasets are used.  We use scPDB for training and validating, and then use PDBbind, Chen (ApoHolo), COACH420 , HOLO4k and SC6K to test. 

You can download dataset from here:

scPDB (http://bioinfo-pharma.u-strasbg.fr/scPDB/)  
PDBbind (http://www.pdbbind.org.cn/download.php)  
Chen (ApoHolo) (http://biomine.cs.vcu.edu/datasets/BindingSitesPredictors/main.html)  
COACH420 (https://github.com/rdk/p2rank-datasets/tree/master/coach420)  
HOLO4k (https://github.com/rdk/p2rank-datasets/tree/master/holo4k)  
SC6K (https://github.com/devalab/DeepPocket)  


#### 2. process
Following [DeepPocket](https://github.com/devalab/DeepPocket) settings , we adopt Fpocket firstly to detect possible potential site center points. Take each point as the center to crop a 32-long protein local cube block. Then we use the libmolgrid to convert 3D local protein structure to multi-channel 3D voxelized grids ($14 \times 65 \times 65 \times 65$), and finally feed the voxelized data into RecurPocket.  
If use your own dataset,
1. run `clean_pdb.py` 
2. run fpocket to get candidate centers
3. run `get_centers.py` to parse fpocket output for candidate centers
4. run `types_and_gninatyper.py` to create gninatypes and types file for CNN input
5. run `rank_pockets.py ` to rerank types input according to CNN score

   


## Pre-trained Models

You can download the RecurPocket model from this [link](https://jbox.sjtu.edu.cn/l/w1wkmK) (code：fpaa), and place the `ckpt` under `RecurPocket/`.

## Test

If you want to test RecurPocket, please run `coach_test.py`.  
Parameter options:  
```
python coach_test.py  [--gpu DEVICE]
                      [--is_dvo COMPUTE_DVO]
                      [--is_mask VOXEL_LEVEL_MASKING]
                      [--iteration RECURRENT_TIMES]
                      [--is_dca COMPUTE_DCA]
```
For example, to test RecurPocket ($\tau=2$) with voxel-level masking on COACH420, run the following command:  
For DCA (Top-n):  
```
python coach_test.py --gpu '0' --is_dvo 0 --is_mask 1 --iteration 3 --is_dca 1  
```
For DCC and DVO:  
```
python coach_test.py --gpu '0' --is_dvo 1 --is_mask 1 --iteration 3 --is_dca 0  
```
If you want to test on your own testset, you need to modify the parameter variable `test_set` and get protein and corresponding labels of your testset.




## Train
If you want to train RecurPocket, please run `train_segmentation.py`.  
Parameter options:  
```
python coach_test.py  [--gpu DEVICE]
                      [--is_cao WHICH_MODEL]
                      [--is_mask VOXEL_LEVEL_MASKING]
                      [--iteration RECURRENT_TIMES]
                      [-b BATCH_SIZE]
                      [-o OUTPUT_DIR]
                      [-e MAX_EPOCH]
```
For example, to train RecurPocket ($\tau=2$) with voxel-level masking, run the following command: 
```
python train_segmentation.py --is_mask 1 --is_cao 4 --iteration 3 --gpu '0,1' -b 10 -o './checkpoint/seg0' -e 100
or
./run.sh
```
